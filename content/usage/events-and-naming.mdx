Events in Sankofa represent user or system actions. Good events are:

- action-oriented
- stable over time
- scoped to one meaningful thing happening

## Good examples

- `signup_completed`
- `checkout_started`
- `payment_failed`
- `invite_accepted`
- `api_request_failed`
- `workspace_created`

## Keep properties for context

Use properties for the dimensions that vary:

- plan
- currency
- screen
- source
- failure_code
- experiment_variant

This keeps the event namespace compact while preserving the detail needed for
analysis.

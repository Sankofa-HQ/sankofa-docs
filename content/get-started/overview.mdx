Sankofa is a closed-source SaaS platform for the full **build → ship → observe → recover** loop. Eight products run on a single engine and a unified set of SDKs, so a flag, a deploy, a crash, and a survey share the same identity, the same cohorts, and the same audit trail.

<Callout tone="note" title="Built for product, engineering, and growth — together">
You don't need separate vendors for analytics, error tracking, feature flags, and OTA. Sankofa replaces the seam between them with a single decision protocol that every SDK speaks.
</Callout>

## How the pieces fit together

Sankofa is organized around three concepts you'll see throughout the docs:

<Steps>
  <Step title="Organizations and projects">
    Every customer has an **organization**. Inside an organization, you create **projects** for each app, service, or environment-isolated workload. A project is the boundary for events, flags, configs, deploys, surveys, RBAC, and billing.
  </Step>
  <Step title="Live and test environments">
    Each project has two environments — `live` and `test` — each with its own API key. The engine resolves the environment from the API key on every request. There is no separate "test project" — the same configuration powers both, so you can promote with confidence.
  </Step>
  <Step title="One handshake, every product">
    On app launch and resume, an SDK calls a single decision endpoint. The engine returns a snapshot of every flag, config, survey trigger, and deploy directive — keyed by cohort, ETag-cached, and replayed on the next crash. This is the seam that makes "one binary, zero glue code" possible.
  </Step>
</Steps>

## SDKs at a glance

The web ships as a **multi-package monorepo** so you only pay for what you use. Every other platform ships as a **single SDK** that bundles all eight products.

<Callout tone="tip" title="SDKs are open source">
The SDKs themselves are open-source and developed in public. The Sankofa platform — engine, dashboard, admin — is proprietary. The open SDKs mean you can audit what we send over the wire and contribute fixes.
</Callout>

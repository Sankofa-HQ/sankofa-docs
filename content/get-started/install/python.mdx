The Sankofa Python SDK is published to PyPI as `sankofa-catch`. Import it as `sankofa_catch`. It captures errors, breadcrumbs, transactions, and ships WSGI and ASGI middleware adapters for any Python web framework.

<Callout tone="note" title="Server-side scope">
The Python SDK focuses on **Catch** — there's no `sankofa.flags.get(...)` or `sankofa.config.get(...)` API today. Dedicated Django / Flask / FastAPI / Celery integrations are not yet shipped — wire those frameworks through the WSGI/ASGI adapters or call the SDK directly from your error handlers.
</Callout>

## Requirements

- **Python 3.9+** (3.11+ recommended)
- **pip 22+**

## 1. Install the package

<CodeBlock lang="bash" code={`pip install sankofa-catch`} />

## 2. Initialize at startup

Call `sankofa_catch.init` once, before any imports that may throw — typically the very first thing in `wsgi.py`, `asgi.py`, or your CLI entry point.

<CodeBlock lang="python" filename="app.py" code={`import os
import sankofa_catch

sankofa_catch.init(
    api_key=os.environ["SANKOFA_KEY"],
    endpoint="https://api.sankofa.dev",
    release=os.environ.get("RELEASE_SHA"),
    environment=os.environ.get("ENV", "production"),
)`} />

<ParameterTable
  params={[

## 3. Wire your framework

<CodeGroup>
<CodeGroupItem tab="flask" label="Flask (WSGI)">
<CodeBlock lang="python" filename="app.py" code={`import os
from flask import Flask
import sankofa_catch
from sankofa_catch.wsgi import SankofaWSGI

sankofa_catch.init(api_key=os.environ["SANKOFA_KEY"], endpoint="https://api.sankofa.dev")

app = Flask(__name__)
app.wsgi_app = SankofaWSGI(app.wsgi_app)

@app.route("/")
def home():
    return "ok"`} />
</CodeGroupItem>
<CodeGroupItem tab="django" label="Django (WSGI)">
<CodeBlock lang="python" filename="wsgi.py" code={`import os
import sankofa_catch
from sankofa_catch.wsgi import SankofaWSGI
from django.core.wsgi import get_wsgi_application

sankofa_catch.init(api_key=os.environ["SANKOFA_KEY"], endpoint="https://api.sankofa.dev")

application = SankofaWSGI(get_wsgi_application())`} />
</CodeGroupItem>
<CodeGroupItem tab="fastapi" label="FastAPI (ASGI)">
<CodeBlock lang="python" filename="main.py" code={`import os
from fastapi import FastAPI
import sankofa_catch
from sankofa_catch.asgi import SankofaASGI

sankofa_catch.init(api_key=os.environ["SANKOFA_KEY"], endpoint="https://api.sankofa.dev")

app = FastAPI()
app.add_middleware(SankofaASGI)

@app.get("/")
async def home():
    return {"ok": True}`} />
</CodeGroupItem>
<CodeGroupItem tab="vanilla" label="Vanilla / scripts">
<CodeBlock lang="python" filename="worker.py" code={`import os
import sankofa_catch

sankofa_catch.init(api_key=os.environ["SANKOFA_KEY"], endpoint="https://api.sankofa.dev")

try:
    do_work()
except Exception as exc:
    sankofa_catch.capture_exception(exc)
    raise`} />
</CodeGroupItem>
</CodeGroup>

## 4. Verify the install

<CodeBlock lang="python" code={`sankofa_catch.capture_message("install_check")`} />

Open [app.sankofa.dev](https://app.sankofa.dev) → **Live events**. The message should appear with a `python` source label and the interpreter version captured under default properties.

<Callout tone="warning" title="Flush on graceful shutdown">
Long-running processes (Celery workers, batch jobs) should call `sankofa_catch.flush()` before exiting. Web frameworks wired through `SankofaWSGI` / `SankofaASGI` flush automatically on shutdown.
</Callout>

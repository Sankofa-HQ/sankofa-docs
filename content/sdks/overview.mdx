Sankofa speaks to your code through SDKs. Every official SDK calls into the same engine using the same two protocols — [event ingestion](/concepts/ingestion-model) and the [decision handshake](/concepts/decision-handshake) — so a flag change, a config publish, or a survey trigger reaches every platform identically.

This page is the bird's-eye view of the SDK matrix. Pick the platform you want and follow its dedicated guide.

## The asymmetric matrix

Web is split into **seven small packages**. Every other platform is **one SDK** that bundles all eight products.

## Common ground across all SDKs

Every official SDK shares the same surface for the things that matter:

<Steps>
  <Step title="Initialize once, as early as possible">
    Every SDK has an `init` (or `initialize`) call that takes an `apiKey` and an optional `endpoint`. Subsequent calls are no-ops. Initialize before you fire any tracking calls.
  </Step>
  <Step title="Track events with `track`">
    Every SDK exposes `track(eventName, properties?)` (mobile/web) or `Track(ctx, Event{...})` (Go) — same shape, same semantics, same wire format.
  </Step>
  <Step title="Identify users with `identify`">
    Every SDK exposes `identify(userId, traits?)` to stitch anonymous traffic onto a known user. The `alias` HTTP transport is shared.
  </Step>
  <Step title="Reset on logout">
    Every SDK exposes `reset()` — rotates the session, clears the cached identity, generates a fresh anonymous ID.
  </Step>
  <Step title="Flush before exit">
    Every SDK exposes `flush()` — drains the queue synchronously. Call it from your shutdown handler in long-running processes.
  </Step>
</Steps>

## Common ground across all client SDKs (web + mobile)

Client-side SDKs (web, Flutter, RN, iOS, Android) share three more things:

- **Offline-first event queue** — events are durable across app restarts and survive network failures.
- **Decision handshake at session start** — one snapshot per session covers Switch, Config, Pulse, A/B Switch.
- **Default property collection** — geo, OS, device, browser, app version, session — added automatically.

Server SDKs (Node, Go, Python, Java) have an in-memory queue, no decision handshake (they fetch flags per call with their own short-TTL cache), and a smaller default-property set focused on runtime + request context.

Session Replay allows you to see exactly how users interact with your Flutter application by recording UI states, touch events, and navigation transitions. Sankofa supports two distinct recording modes to balance detail with performance and bandwidth.

## Core Concepts

### Recording Modes

| Mode | Description | Best For |
| :--- | :--- | :--- |
| `wireframe` | Captures the UI structure and layout using lightweight JSON blueprints. | High performance, low bandwidth, default recording. |
| `screenshot` | Captures actual pixel-perfect snapshots of the screen at a target FPS. | High-fidelity debugging of visual bugs or complex animations. |

### Dynamic Triggers

Sankofa can automatically switch from `wireframe` to `screenshot` mode when critical events occur (e.g., a checkout error or an app crash). This ensures you have high-fidelity data exactly when you need it while remaining efficient during normal usage.

## Setup

### 1. Wrap your App

To capture the UI, you must wrap your root application widget with `SankofaReplayBoundary`. This provides the necessary coordinate system and repaint boundaries for the recording engine.

```dart
import 'package:sankofa_flutter/sankofa_flutter.dart';

void main() {
  runApp(
    SankofaReplayBoundary(
      child: MyApp(),
    ),
  );
}
```

### 2. Initialize with Replay

Enable session replay in your `Sankofa.init` call. You can customize the default mode and frame rate.

```dart
await Sankofa.instance.init(
  apiKey: 'YOUR_PROJECT_API_KEY',
  endpoint: 'https://sankofa.your-domain.com',
  enableSessionReplay: true,
  replayMode: SankofaReplayMode.wireframe, // Default mode
  replayFps: 1, // Target FPS for screenshot mode
);
```

### 3. Track Navigation

To see screen names and transitions in your replays, add the `SankofaNavigatorObserver` to your `MaterialApp`.

```dart
MaterialApp(
  navigatorObservers: [
    SankofaNavigatorObserver(),
  ],
  // ...
)
```

## Privacy & Redaction

### Masking Sensitive Components

Use the `SankofaMask` widget to redact sensitive information (like passwords, PII, or credit card numbers). When recording, the masked area will be replaced with a solid black box.

```dart
SankofaMask(
  child: Text('Sensitive User Data'),
)
```

### High-Fidelity Triggers

You can manually trigger a temporary period of high-fidelity recording if you detect a critical state in your app logic.

```dart
// Switch to screenshot mode for the next 30 seconds
Sankofa.instance.triggerHighFidelityMode(Duration(seconds: 30));
```

## Advanced Configuration

### Remote Configuration

Sankofa remotely synchronizes your replay configuration on initialization. The following settings can be managed directly from the **Sankofa Dashboard**, overriding any local defaults:

- **Sampling**: Control the percentage of standard sessions to record (e.g., only record 10% of users).
- **High Fidelity Triggers**: Define a list of event names (e.g., `error`, `purchase_failed`) that switch the SDK to high-detail screenshot mode.
- **Burst Duration**: Set how many seconds of high-fidelity data to capture after a trigger occurs.
